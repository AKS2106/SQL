# Healthcare Readmission Batch Pipeline

A PySpark batch pipeline that processes 101,766 hospital patient
records to identify readmission patterns, flag high-risk patients,
and produce a daily summary report for clinical operations teams.

---

## Business Problem

Hospitals in the US lose billions annually to preventable 30-day
readmissions. The CMS Hospital Readmissions Reduction Program
penalises hospitals financially for excess readmissions.

This pipeline processes raw patient admission records daily to:
- Identify patients readmitted within 30 days
- Categorise patients by risk tier (High / Medium / Low)
- Surface departments and age groups with highest readmission rates
- Give the clinical ops team actionable data every morning

---

## Architecture

```
Raw CSV (S3 / local)
       ↓
  ingest.py        → validates schema, replaces nulls, saves Parquet
       ↓
  quality.py       → 6 data quality checks, flags bad records
       ↓
  transform.py     → adds readmission flag, risk tier, age groups
       ↓
Parquet output     → partitioned by risk_tier
       ↓
  SQL / Athena     → ad-hoc analysis queries
```

## What This Pipeline Produces

| Output | Description |
|---|---|
| Readmission flag | 1 if patient readmitted within 30 days, 0 otherwise |
| Risk tier | High (7+ days), Medium (4-6 days), Low (1-3 days) |
| Age group | Cleaned from raw bracket format into readable labels |
| LOS category | Short / Medium / Long length of stay |
| Quality report | Pass/Fail for 6 checks printed on every run |

## Key Technical Decisions

**Parquet over CSV for processed data**
Spark reads only the columns it needs from Parquet — columnar format.
For a 50-column dataset this is significantly faster than CSV which
reads all columns every time.

**Explicit schema — no inferSchema**
inferSchema reads the entire file twice and can guess types wrong.
Defining schema explicitly is production standard and catches
upstream type changes immediately.

**coalesce(4) before writing**
Default Spark shuffle creates too many small files. coalesce(4)
reduces to 4 output files — right-sized for a 100k row dataset
targeting ~128MB per file.

**partitionBy("risk_tier") on output**
Downstream queries almost always filter by risk tier. Partitioning
means Spark skips non-matching folders — partition pruning reduces
scan cost significantly at scale.

**PERMISSIVE read mode with "?" replacement**
This dataset uses "?" for missing values. PERMISSIVE mode keeps all
rows and the "?" replacement converts them to proper NULL so all
downstream aggregations handle missing data correctly.

## How to Run

**Requirements**
- Python 3.8+
- Java 8 or Java 11 (not Java 17)
- Windows: also requires winutils.exe (see setup notes)

**Install dependencies**
```bash
pip install -r requirements.txt
```

**Download the dataset**
```
https://raw.githubusercontent.com/14Richa/Patient-Readmission-Analysis/main/diabetic_data.csv
```
Save it to: `data/raw/diabetic_data.csv`

**Run the full pipeline**
```bash
python main.py
```

**Or run individual steps**
```bash
python src/ingest.py      # Step 1: read and clean raw CSV
python src/quality.py     # Step 2: run data quality checks
python src/transform.py   # Step 3: add business logic
```

-- Q1: Readmission rate by risk tier
SELECT
    risk_tier,
    COUNT(*)                                      AS total_patients,
    SUM(is_readmission_30d)                       AS readmissions,
    ROUND(100.0 * SUM(is_readmission_30d)
          / COUNT(*), 1)                          AS readmit_rate_pct
FROM admissions
GROUP BY risk_tier
ORDER BY readmit_rate_pct DESC;

-- Q2: Readmission rate by age group
SELECT
    age_group,
    COUNT(*)                                      AS patients,
    ROUND(100.0 * SUM(is_readmission_30d)
          / COUNT(*), 1)                          AS readmit_rate_pct
FROM admissions
GROUP BY age_group
ORDER BY readmit_rate_pct DESC;

-- Q3: Top 5 medical specialties by readmission rate
-- Only specialties with 100+ patients for statistical significance
WITH specialty_stats AS (
    SELECT
        medical_specialty,
        COUNT(*)                                  AS patients,
        ROUND(100.0 * SUM(is_readmission_30d)
              / COUNT(*), 1)                      AS readmit_rate_pct
    FROM admissions
    WHERE medical_specialty IS NOT NULL
    GROUP BY medical_specialty
    HAVING COUNT(*) >= 100
)
SELECT *
FROM specialty_stats
ORDER BY readmit_rate_pct DESC
LIMIT 5;

-- Q4: Patients with multiple admissions (high risk for readmission)
SELECT
    patient_nbr,
    COUNT(*)                                      AS total_visits,
    SUM(is_readmission_30d)                       AS readmit_count,
    ROUND(AVG(time_in_hospital), 1)               AS avg_los_days,
    MAX(num_medications)                          AS max_medications
FROM admissions
GROUP BY patient_nbr
HAVING COUNT(*) > 1
ORDER BY total_visits DESC
LIMIT 20;

-- Q5: Average medications by risk tier and readmission status
SELECT
    risk_tier,
    is_readmission_30d,
    COUNT(*)                                      AS patients,
    ROUND(AVG(num_medications), 1)                AS avg_medications,
    ROUND(AVG(num_lab_procedures), 1)             AS avg_lab_procedures
FROM admissions
GROUP BY risk_tier, is_readmission_30d
ORDER BY risk_tier, is_readmission_30d DESC;
